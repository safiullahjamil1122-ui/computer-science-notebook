# Contributing

Thanks for helping improve this repository.

## What to contribute

- Fix factual errors in the documentation.
- Improve examples, diagrams, and explanations.
- Tighten markdown formatting and navigation.
- Add missing references or practice problems where they add value.

## Standards

- Keep changes focused and easy to review.
- Preserve the existing documentation style.
- Prefer clear, concise language over filler.
- Update related links or tables of contents when you rename or move content.

## Suggested workflow

- Open an issue before large structural changes.
- Keep commits small and logically grouped.
- Verify markdown links after editing any README or topic file.
- Run the markdown audit workflow before opening a pull request.
